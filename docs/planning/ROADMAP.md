# 项目路线图

> Phase 0-8 的分阶段路线图，含详细步骤、工作量预估和关键风险。

---

## 阶段总览

```
Phase 0: 基础设施 & PoC        ← 在 agent-engineering-practices 中
Phase 1: 产品需求分析           ← 试点: 站点管理模块
Phase 2: UX/UI 设计
Phase 3: 架构设计（⛔ 阻断门禁）
Phase 4: 工作流设计（可选）
Phase 5: 前端实现 + 评估迭代
Phase 6: 后端开发
Phase 7: 质量保障
Phase 8: 部署 & 运维
```

## Phase 0: 基础设施 & PoC

**在哪里**: agent-engineering-practices 项目中
**目标**: 搭建 Agent + Skills 工具链，用站点管理模块做端到端验证

| 步骤 | 内容 | Agent 辅助度 | 工作量 |
|------|------|-------------|--------|
| 0.1 | 继承治理机制（CORRECTIONS、STANDARDS、宪法） | 中 | 2-3 天 |
| 0.2 | Skills 开发框架 + 模板 | 低（需人工设计） | 2-3 天 |
| 0.3 | 治理层 Skills（4 个） | 中 | 3-4 天 |
| 0.4 | 产品/设计层 Skills（6 个） | 中 | 4-6 天 |
| 0.5 | 工程层 Skills（6 个） | 中 | 3-5 天 |
| 0.6 | 质量/评估层 Skills（4 个） | 中 | 3-4 天 |
| 0.7 | MCP 工具服务（6 个工具） | 中 | 3-4 天 |
| 0.8 | PoC 端到端验证（站点管理列表页） | 高 | 3-4 天 |
| 0.9 | 团队培训材料 | 高 | 2-3 天 |
| **小计** | | | **25-36 天（约 4-6 周）** |

**PoC 验收标准**:
- [ ] Agent 能正确执行"需求分析 → 架构设计 → 前端生成"的 3 阶段流程
- [ ] 阻断性门禁有效工作（跳过架构确认时被拦截）
- [ ] 生成的代码通过 TypeScript 编译 + ESLint
- [ ] 术语扫描检测到不一致时能正确报告
- [ ] Session 启动时能正确加载上下文和 CORRECTIONS 意识
- [ ] 团队成员能独立使用 Skills 进行开发

**关键风险**: Agent SDK 学习曲线（团队无 SDK 经验，需 Phase 0 充分培训）

## Phase 1: 产品需求分析

**试点模块**: 站点管理
**目标**: 完整的需求分析

| 步骤 | 内容 | 说明 |
|------|------|------|
| 1.1 | 需求拆解（L1-L4） | 使用 requirement-decomposition Skill |
| 1.2 | User Story 编写 | 使用 user-story-writing Skill |
| 1.3 | 用户确认 ⛔ | User Story 须与真实用户确认 |
| 1.4 | 反向影响审查 | 检查上游模块 architecture.md |

**关键输出**: requirements.md + user-stories.md
**工作量**: 3-4 天

**关键风险**: 需求持续变化（与用户沟通处于早期）→ 模块独立闭环 + 渐进式确认

## Phase 3: 架构设计（⛔ 阻断门禁）

| 步骤 | 内容 | 说明 |
|------|------|------|
| 3.1 | 实体三问分析 | Q1 字段 / Q2 创建方式 / Q3 副作用 |
| 3.2 | 数据库 Schema 草案 | PostgreSQL ENUM + CREATE TABLE + 索引 |
| 3.3 | 跨模块 ER 图更新 | cross-module-erd.md |
| 3.4 | 跨模块数据流定义 | 下游消费者 + 触发机制 + 数据契约 |
| 3.5 | 聚合接口预分析 | 看板/仪表盘需要的聚合查询 |
| 3.6 | Placeholder 标记 | 共享功能标记为 `⚠️ Phase N Dependency` |
| 3.7 | API 设计 | RESTful + JSON 示例（P10 模式） |
| 3.8 | 用户确认 ⛔⛔ | **阻断门禁**: 未确认不可进入实现 |

**关键输出**: architecture.md + cross-module-erd.md 更新
**工作量**: 3-4 天

**关键风险**: 跨模块数据一致性 → ER 图约束 + cross-domain-validator
