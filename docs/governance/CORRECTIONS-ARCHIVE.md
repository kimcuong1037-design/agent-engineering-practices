# 纠偏完整档案

> **文档类型**: 活文档（随项目演进追加）
> **与 CORRECTIONS.md 的关系**: CORRECTIONS.md 是摘要索引（§1 模式表 + §2 时间线），本文件存储每条纠偏的完整上下文和根因分析。
> **使用规则**: 新增纠偏时，先更新 CORRECTIONS.md，再在本文件追加完整记录。

---

## 记录格式模板

```
### [P编码] [标题]

**日期**: YYYY-MM-DD
**触发场景**: [用户在什么情境下指出了问题]
**问题描述**: [Agent 输出了什么错误/不规范的内容]
**根因分析**: [为什么会出现这个问题，底层原因是什么]
**纠偏措施**: [应该怎么做，具体规则是什么]
**影响范围**: [哪些 Skill / 文档 / 代码需要同步更新]
**关联模式**: [P编码，如已更新到 CORRECTIONS.md §1]
```
